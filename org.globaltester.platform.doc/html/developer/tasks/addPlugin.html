Add a new Plug-in

Whenever additional functionality is only optional for GlobalTester and can be decoupled from the existing plug-ins it is a good idea to start a new plug-in project. You will find some hints when to start a new plug-in and when to add to an existing one in our development guidelines.

Create plug-in project

To create a new plug-in project from scratch start the "New Project" wizard. In Eclipse, select "Package Explorer" view "New > Project" and there use "Plug-in Project" and click "Next".

In the following screens fill in the project name, plug-in ID and related data. You will find several hints regarding the values to use here in our development guidelines Again.

When the basic plug-in structure is created you can start to implement the code just as ordinary Eclipse plug-in development. Remember to add a second plug-in project containing a fragment to your plug-in which includes all your tests.

For your plug-in to be available during execution and Buckminster provisioning it, it must be added to a feature that is referenced from the Buckminster entry point. Add it to the features where your plug-in fits or add a new feature if none fits.
