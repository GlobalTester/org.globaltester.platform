Get an IDE

As GlobalTester is based on Eclipse RCP we also use Eclipse as our IDE. You can either get a recent package from eclipse.org and install the required plug-ins on your own or get a prepared Eclipse installation from HJP Consulting which already includes all required and suggested plug-ins, see section Get prepared Eclipse installation for details.

To set it up on your own go get a recent package from http://www.eclipse.org/downloads/. Just unzip it to a location of your choice and fire up the executable inside and install the required plug-ins as listed below.

Required plug-ins

To get the code and handle it you will need at least the following plug-ins:

  • PDE (this may be already included, depending on the package you chose to download)
  • Buckminster
  • Subversion integration (choose whichever you want, we use Subversive)

Optional plug-ins

  • WebPageEditor (for editing documentation)
  • JUnit (for unit testing)
  • SWTBot (for testing UI)
  • WindowBuilder (for easier design of UI)

Get prepared Eclipse installation

You can get a zipped Eclipse installation already including all mentioned required and optional plug-ins from Y:/Java Tools/Eclipse/eclipse-SDK-3.6.2-gt-dev-default.zip. Currently this is only available for win32 and only accessible from within HJP Consultings private network.

We use little out-of-date version of Eclipse, which is the oldest one GlobalTester states to support. By this we make sure that you don't get confused by hints in the online help or similar that won't work when compiling against the fixed target platform.
