Java and Eclipse Versions

GlobalTester is supposed to be able to be used with the current and preceding Eclipse and Java version. Right now Eclipse 3.6 (Helios) and Java 1.6 should be used. Any new plug-ins, Bundles, Fragments, etc. should satisfy this condition. Dependencies to Eclipse 3.7 (Indigo) or Java 7 should be avoided.

SVN Repository Structure

The directory "Trunk" should be underneath the projects. This way ensures that versions can be tagged for the single plug-ins and no ambiguous tags can appear in the whole repository. If you share a project, please make sure you follow the intructions in our help tasks.

Launch Configurations

The launch configuration "GT3_RCP" should exactly represent the state, which is in accordance with the completed Jenkins build. If you want to use your own plug-ins, please use the launch configuration "GT3_SDK". This one starts GlobalTester with all plug-ins in your workspace enabled in a full-fledged Eclipse IDE. Your own plug-in will be integrated to the product by adding it to the according feature. Please make sure you use an Eclipse target platform and not the GlobalTester target platform.
